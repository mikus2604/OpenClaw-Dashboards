# Contributing to OpenClaw Dashboards

Thanks for helping make this list more comprehensive!

## How to Add a Dashboard

1. **Fork** this repository
2. **Edit** `README.md` — add the dashboard to the most appropriate category
3. **Open a Pull Request** with a brief description of the project


## Eligibility Requirements

- ✅ Specifically built for OpenClaw (not a generic dashboard)
- ✅ Open-source with a public GitHub repository
- ✅ At least 5 meaningful feature bullet points
- ✅ A real screenshot (not a placeholder or badge)
- ✅ Working install instructions

## Categories

Place the dashboard in the correct section:

| Category | When to use |
|---|---|
| **Multi-Agent Orchestration** | Focuses on running teams of agents, workflows, approvals, multi-org |
| **Web Dashboards** | Browser-based UI for monitoring/managing a single agent or setup |
| **Terminal Dashboards** | CLI/TUI that runs in the terminal |
| **Mobile Apps** | Native Android/iOS apps |

## Screenshot Tips

- Use a direct link to an image in the repo via `raw.githubusercontent.com`
- Or use a GitHub CDN link (`https://github.com/user-attachments/assets/...`)
- Image should be at least 800px wide for good display on GitHub
- Animated GIFs are welcome for showing key interactions

## Questions?

Open an [issue](https://github.com/mikus2604/OpenClaw-Dashboards/issues) if you're unsure about anything.
